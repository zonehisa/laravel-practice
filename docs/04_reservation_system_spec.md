# 染め物屋高橋 予約管理一元化システム 要件定義書

## 1. プロジェクト概要
本システムは、盛岡手づくり村に出店する「染め物屋高橋」の体験イベント予約管理を効率化し、一元管理を実現することを目的とする。中央課題における「データの散在による管理の煩雑さ」と「ヒューマンエラーのリスク」を解決する役割を担う。

- **現状の課題 (As-Is):**
  - 予約経路（じゃらん、アソビュー、自社HP、電話等）が複数存在し、情報が散在している。
  - 各経路からの予約をExcelへ手動で転記しており、手間と時間がかかる。
  - 手動管理により、ダブルブッキングが発生するリスクを常に抱えている。
- **導入後の理想 (To-Be):**
  - すべての予約情報が一つのシステムに集約され、リアルタイムで状況を把握できる。
  - 手動転記作業が不要になり、予約管理業務が大幅に効率化される。
  - ダブルブッキングがシステム的に防止され、顧客満足度が向上する。
- **対象ユーザー:** 染め物屋高橋の予約管理担当者

## 2. ユーザーストーリー
「染め物屋高橋の予約管理担当者として、【すべての予約を一元管理したい】ので、【ダブルブッキングを防ぎ、予約状況をカレンダーで簡単に把握できるようにしてほしい】」

## 3. 機能要件

| 優先度 | 機能分類 | 機能概要 |
| :--- | :--- | :--- |
| **Must** | カレンダー表示 | 予約状況を視覚的に把握できる月表示および週表示のカレンダー機能。 |
| **Must** | 予約の手動登録 | 電話などオフラインでの予約をシステムに直接登録できる機能。 |
| **Must** | 予約詳細・編集 | 予約内容の確認、変更、キャンセル処理を行える機能。 |
| Could | メール自動登録 | 各予約サイトから届く予約メールの内容をシステムが自動で解析し、カレンダーに登録する機能。 |
| Could | アラート機能 | ダブルブッキングの可能性がある予約登録時に、警告を表示する機能。 |
| Could | 確認メール自動送信 | 予約登録・変更・キャンセル時に、顧客へ確認メールを自動で送信する機能。 |
| Could | 統計表示 | 月別予約数、キャンセル率、予約経路別の割合などをグラフで可視化する機能。 |

## 4. データ要件
システム内で管理・保存すべきデータ項目は以下の通り。

- 予約ID
- 予約日時（開始・終了）
- 体験プログラム名
- 予約者名
- 連絡先（電話番号、メールアドレス）
- 予約人数
- 予約経路（じゃらん/アソビュー/HP/電話など）
- 予約状態（予約済み/キャンセル/完了）
- 備考
